# 正则表达式学习笔记

## 1. 正则表达式是什么

正则表达式是一套用于匹配字符串的规则。

常见用途：

- 判断文本是否符合某种格式
- 从文本中提取目标内容
- 对文本做替换
- 按规则切分字符串

例如：

- 校验手机号
- 提取邮箱
- 匹配 Markdown 图片
- 匹配日期、数字、文件名

---

## 2. Python 中常用的 `re` 方法

```python
import re
```

常见方法：

- `re.search(pattern, text)`：在整个字符串中查找第一个匹配
- `re.match(pattern, text)`：只从字符串开头开始匹配
- `re.fullmatch(pattern, text)`：要求整个字符串完全匹配
- `re.findall(pattern, text)`：返回所有匹配结果
- `re.finditer(pattern, text)`：返回所有匹配对象
- `re.sub(pattern, repl, text)`：替换
- `re.split(pattern, text)`：按规则切分

示例：

```python
import re

text = "my phone is 12345 and 67890"
result = re.findall(r"\d+", text)
print(result)
```

输出：

```python
['12345', '67890']
```

## 3. 普通字符匹配

普通字符一般就表示匹配它本身。

```python
re.search(r"cat", "my cat is here")
```

会匹配字符串 `cat`。

## 4. 常见元字符

元字符是具有特殊含义的字符。

| 元字符 | 含义 |
| --- | --- |
| `.` | 任意单个字符 |
| `^` | 字符串开头 |
| `$` | 字符串结尾 |
| `*` | 前一个内容出现 0 次或多次 |
| `+` | 前一个内容出现 1 次或多次 |
| `?` | 前一个内容出现 0 次或 1 次 |
| `[]` | 字符集合 |
| `()` | 分组 |
| `|` | 或 |
| `\` | 转义 |

---

## 5. 点号 `.` 的含义

```python
.
```

表示匹配任意一个字符，通常不包含换行。

```python
import re

print(re.findall(r"a.c", "abc aXc"))
```

输出：

```python
['abc', 'aXc']
```

---

## 6. 字符集合 `[]`

字符集合表示当前位置可以是某几个字符之一。

```python
[abc]
```

表示匹配：

- `a`
- `b`
- `c`

示例：

```python
import re

print(re.findall(r"[abc]", "apple cab"))
```

常见写法：

- `[0-9]`：一个数字
- `[a-z]`：一个小写字母
- `[A-Z]`：一个大写字母
- `[a-zA-Z0-9]`：一个字母或数字

取反写法：

```python
[^0-9]
```

表示匹配不是数字的任意一个字符。

## 7. 预定义字符类

这些写法非常常用：

| 写法 | 含义 |
| --- | --- |
| `\d` | 数字 |
| `\D` | 非数字 |
| `\w` | 字母、数字、下划线 |
| `\W` | 非字母数字下划线 |
| `\s` | 空白字符 |
| `\S` | 非空白字符 |

示例：

```python
import re

print(re.findall(r"\d+", "a12b34"))
```

输出：

```python
['12', '34']
```

## 8. 数量词

数量词决定前面的内容匹配多少次。

| 写法 | 含义 |
| --- | --- |
| `*` | 0 次或多次 |
| `+` | 1 次或多次 |
| `?` | 0 次或 1 次 |
| `{n}` | 恰好 n 次 |
| `{m,n}` | m 到 n 次 |
| `{m,}` | 至少 m 次 |

示例：

```python
import re

print(re.findall(r"\d{3}", "1234 567 89"))
```

输出：

```python
['123', '567']
```

说明：

- `\d{3}` 表示连续 3 个数字

## 9. 贪婪匹配和非贪婪匹配

默认情况下，很多数量词是贪婪的，意思是尽可能多匹配。

例如：

- `.*`
- `.+`
- `\d+`

非贪婪匹配是在贪婪量词后面加 `?`，表示尽可能少匹配。

例如：

- `.*?`
- `.+?`
- `\d+?`

### 贪婪匹配示例

```python
import re

text = "a123b456b"
m = re.search(r"a.*b", text)
print(m.group())
```

输出：

```python
a123b456b
```

原因：

- `.*` 会尽可能多吃字符
- 它会一直匹配到最后一个还能让整个表达式成立的 `b`

### 非贪婪匹配示例

```python
import re

text = "a123b456b"
m = re.search(r"a.*?b", text)
print(m.group())
```

输出：

```python
a123b
```

原因：

- `.*?` 会尽量少匹配
- 遇到第一个可以让整体匹配成功的 `b` 就停

### 一个更直观的例子

```python
text = "<div>hello</div><div>world</div>"
```

贪婪：

```python
re.search(r"<div>.*</div>", text).group()
```

结果：

```python
<div>hello</div><div>world</div>
```

非贪婪：

```python
re.search(r"<div>.*?</div>", text).group()
```

结果：

```python
<div>hello</div>
```

## 10. `.*?` 和 `*?` 的区别

这是初学者很容易混淆的地方。

### `.*?` 是合法写法

拆开理解：

- `.`：任意单个字符
- `*`：前面的 `.` 出现 0 次或多次
- `?`：把 `*` 变成非贪婪

所以：

```python
.*?
```

表示：

任意字符出现 0 次或多次，但尽量少匹配。

### `*?` 单独写通常是非法的

原因：

- `*` 是数量词
- 数量词必须修饰前面的某个内容
- 单独写 `*?`，前面没有对象可以重复

所以会报类似错误：

```python
nothing to repeat
```

## 11. 开头和结尾匹配

| 写法 | 含义 |
| --- | --- |
| `^` | 开头 |
| `$` | 结尾 |

示例：

```python
import re

print(re.search(r"^\d{11}$", "13800138000"))
```

表示：

- 整个字符串从头到尾必须是 11 位数字

---

## 12. 单词边界

| 写法 | 含义 |
| --- | --- |
| `\b` | 单词边界 |
| `\B` | 非单词边界 |

示例：

```python
import re

print(re.findall(r"\bcat\b", "cat scatter catalog"))
```

结果只会匹配独立单词 `cat`，不会匹配 `scatter` 或 `catalog` 中的一部分。

---

## 13. 分组 `()`

分组可以把多个表达式看成一个整体，也可以提取局部匹配结果。

示例：

```python
import re

m = re.search(r"(\d{4})-(\d{2})-(\d{2})", "2026-06-04")
print(m.group(0))
print(m.group(1))
print(m.group(2))
print(m.group(3))
```

说明：

- `group(0)`：整体匹配结果
- `group(1)`：第一组
- `group(2)`：第二组
- `group(3)`：第三组

---

## 14. 非捕获分组

写法：

```python
(?:...)
```

作用：

- 参与分组
- 但不作为单独的捕获组保存

例如：

```python
(?:http|https)://
```

表示匹配：

- `http://`
- `https://`

## 15. 或 `|`

```python
cat|dog
```

表示匹配：

- `cat`

示例：

```python
import re

print(re.findall(r"cat|dog", "cat and dog"))
```

输出：


## 16. 转义 `\`

有些字符在正则中有特殊意义，如果要匹配它本身，需要转义。

例如：

- `\.`：匹配真正的点号 `.`
- `\(`：匹配左括号
- `\)`：匹配右括号
- `\[`：匹配左中括号
- `\]`：匹配右中括号

例如：

```python
a\.png
```

表示匹配真正的文件名 `a.png`，而不是“a + 任意字符 + png”。

## 17. 为什么 Python 里常写 `r"..."` 

Python 里的正则通常写成原始字符串：

```python
r"\d+"
```

原因是反斜杠在普通字符串里本身也有特殊意义。

例如下面两种写法效果相同：

```python
"\\d+"
r"\d+"
```

但第二种更直观，所以更常用。

## 18. `match`、`search`、`fullmatch` 的区别

示例：

```python
import re

text = "abc123"
```

### `re.match()`

```python
re.match(r"\d+", text)
```

说明：

- 只从开头匹配
- 这里会失败，因为开头是 `abc`

### `re.search()`

```python
re.search(r"\d+", text)
```

说明：

- 在整个字符串中查找
- 这里可以找到 `123`

### `re.fullmatch()`

```python
re.fullmatch(r"\d+", text)
```

说明：

- 要求整个字符串完全是数字
- 这里也会失败

---

## 19. `findall` 和 `finditer` 的区别

### `findall`

- 直接返回匹配结果列表
- 适合快速查看结果

### `finditer`

- 返回迭代器
- 每一项都是 `match` 对象
- 可以拿到位置、分组等详细信息

示例：

```python
import re

for m in re.finditer(r"\d+", "a12b34"):
    print(m.group(), m.span())
```

## 22. 常见实用正则示例

### 匹配整数

```python
r"\d+"
```

### 匹配手机号（简单版）

```python
r"1\d{10}"
```

### 匹配邮箱（简单版）

```python
r"[\w.-]+@[\w.-]+\.\w+"
```

### 匹配日期

```python
r"\d{4}-\d{2}-\d{2}"
```

例如可匹配：

```python
2026-06-04
```

### 匹配 Markdown 图片

```python
r"!\[.*?\]\(.*?\)"
```

## 23. Markdown 图片上下文匹配案例

下面这类正则：

```python
r"!\[.*?\]\(.*?" + re.escape(image_file) + r".*?\)"
```

作用是匹配类似下面这种 Markdown 图片语法：

```md
![描述](images/demo.png)
```

它的思路是：

1. 先匹配 `![`
2. 用 `.*?` 匹配图片描述
3. 再匹配 `](`
4. 再匹配路径中的内容
5. 用 `re.escape(image_file)` 精确匹配目标图片文件名
6. 最后匹配右括号 `)`

如果代码再拿到匹配位置的 `start` 和 `end`，就可以向前、向后截取固定长度的文本，作为图片的上下文。

## 24. 正则常见坑

- `.*` 很容易匹配过头
- 忘记转义 `.`、`(`、`)`、`[`、`]`
- 把 `match()` 当成全局查找
- 注释写“找所有”，代码却只用了 `search()`
- 忘记 `*`、`+`、`?` 都是修饰前一个对象
